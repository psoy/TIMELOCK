# TIMELOCK 사용자 페르소나 및 가치제안 캔버스

## 1. 사용자 페르소나 (User Personas)

### 페르소나 1: 독서실 수험생 김지수 (Primary Persona) ⭐

![Persona Type: Primary Target - 독서실/도서관 수험생]

#### 기본 정보
- **이름**: 김지수
- **나이**: 24세
- **직업**: 공무원 시험 준비생 (3수)
- **학력**: 대학 졸업 (경영학과)
- **거주지**: 서울 노원구
- **일상**: 매일 오전 8시~오후 10시 독서실 출퇴근

#### 인구통계학적 특성
- **소득**: 부모님 지원 + 아르바이트 (월 80만원)
- **기술 친화도**: 높음 (스마트폰, 태블릿 능숙)
- **선호 기기**: 아이패드 (필기), 갤럭시 스마트폰
- **앱 사용**: Notion (정리), Forest (집중), 유튜브 (강의)

#### 성격 및 행동 특성
- **MBTI**: ISTJ (신중형, 계획적, 책임감)
- **학습 스타일**: 체계적 계획 선호, 루틴 중시, 데이터 기반 개선
- **성격**: 완벽주의 성향, 자기 관리 엄격, 목표 지향적
- **스트레스 요인**: 계획대로 안 될 때, 시간 낭비, 성적 부진

#### 하루 일과 (Typical Day)
```
07:00 - 기상, 아침 식사
08:00 - 독서실 도착, 하루 계획 수립 (종이 플래너)
08:30 - 1교시: 행정법 (90분)
10:00 - 휴식 10분
10:10 - 2교시: 행정학 (90분)
11:40 - 점심 (40분)
12:20 - 3교시: 영어 (90분)
14:00 - 휴식 10분 (커피)
14:10 - 4교시: 한국사 (90분)
15:40 - 휴식 20분
16:00 - 5교시: 문제 풀이 (120분)
18:00 - 저녁 식사 (60분)
19:00 - 6교시: 복습 및 암기 (90분)
20:30 - 7교시: 오답 정리 (60분)
21:30 - 귀가, 일일 복습
23:00 - 취침
```

#### 목표 (Goals)
1. **단기 목표**: 올해 안에 공무원 시험 최종 합격
2. **학습 목표**: 하루 10시간 순수 학습 시간 확보
3. **효율 목표**: 계획 대비 실행률 80% 이상 달성
4. **습관 목표**: 매일 같은 시간에 같은 과목 공부 (루틴 형성)

#### 고충점 (Pain Points)
1. **시간 관리의 어려움**
   - "계획은 세웠는데 실제로 얼마나 공부했는지 모르겠어요"
   - 휴대폰을 보다가 시간이 훌쩍 지나감
   - 50분 계획이 실제로는 30분만 집중함

2. **조용한 환경의 알림 제약**
   - "독서실에서 타이머 소리가 나면 눈치가 보여요"
   - Forest 앱의 새소리가 민망함
   - 진동도 옆 사람에게 들릴까 걱정됨

3. **계획과 실행의 괴리**
   - "플래너에 예쁘게 적어놨는데 실제로는 잘 안 지켜져요"
   - 계획 세우는 데만 30분 소모
   - 계획대로 안 되면 죄책감과 스트레스

4. **분석 및 개선의 어려움**
   - "어제보다 나아졌는지 알 수 없어요"
   - 어떤 시간대에 집중이 잘 되는지 모름
   - 데이터 없이 감으로만 공부 시간 조절

5. **멀티 도구 사용의 피로**
   - 종이 플래너 (계획) + Forest (타이머) + 스프레드시트 (기록)
   - 각각 따로 관리하느라 시간 낭비
   - 도구 간 데이터 연동 안 됨

#### 니즈 (Needs)
1. **조용한 알림**: 소리 없이도 확실히 인지되는 타이머
2. **통합 도구**: 계획-실행-분석이 하나의 앱에서
3. **데이터 분석**: 객관적인 학습 시간 및 패턴 파악
4. **간단한 UX**: 공부 시작 전 10초 안에 세팅 완료
5. **동기부여**: 실행률 향상을 눈으로 확인

#### 사용 시나리오 (Use Scenarios)
**시나리오 1: 아침 독서실 도착 후**
- 독서실 자리에 앉아 TIMELOCK 앱 실행
- 오늘의 시간 블록 확인 (전날 밤에 미리 설정)
- "행정법 90분" 블록 터치 → 타이머 시작
- 화면 플래시 알림 설정 (무음 모드)
- 집중 시작

**시나리오 2: 타이머 종료 시**
- 90분 후 화면이 밝게 깜박이며 알림
- 소리 없이 시각적으로만 알림 → 주변 방해 없음
- "완료" 버튼 터치 → 다음 블록 자동 시작 준비
- 10분 휴식 블록 시작

**시나리오 3: 저녁 귀가 전 복기**
- 오늘의 리포트 확인
- 계획: 10시간 / 실행: 8시간 30분 (85%)
- 어떤 블록에서 집중이 흐트러졌는지 확인
- 내일 계획 수정 (점심 후 집중도 낮음 → 블록 시간 단축)

#### 선호하는 기능
- ✅ 무음 화면 플래시 알림
- ✅ 타임블로킹 자동 반복 (매일 같은 루틴)
- ✅ 실행률 통계 (일/주/월)
- ✅ 집중 패턴 히트맵
- ✅ 간단한 UI (버튼 3개 이내)

#### 사용하지 않을 것 같은 기능
- ❌ 감정 일기
- ❌ 게이미피케이션 (나무 키우기 등)
- ❌ 복잡한 설정
- ❌ SNS 공유 기능

#### 인용구 (Quotes)
> "독서실에서 타이머 소리 나면 정말 민망해요. 하지만 타이머 없이는 시간 가는 줄 몰라요."

> "계획표는 예쁘게 만들었는데, 실제로 얼마나 지켰는지 알 수 없어요. 숫자로 보고 싶어요."

> "Forest는 좋은데, 소리가 나서 독서실에서 못 쓰겠어요. 진동도 옆 사람한테 들려요."

> "하루가 끝나고 '오늘 뭐했지?' 싶을 때가 많아요. 시간을 정확히 추적하고 싶어요."

### 페르소나 2: 프리랜서 개발자 박준호 (Secondary Persona)

#### 기본 정보
- **이름**: 박준호
- **나이**: 31세
- **직업**: 프리랜서 웹 개발자
- **거주지**: 서울 강남구
- **근무 환경**: 홈 오피스 + 카페 (주 3회)

#### 인구통계학적 특성
- **소득**: 월 400-600만원 (프로젝트별)
- **기술 친화도**: 매우 높음
- **선호 기기**: 맥북 프로, 아이폰
- **앱 사용**: Notion, Toggl, Slack, GitHub

#### 성격 및 행동 특성
- **MBTI**: INTJ (전략가형)
- **업무 스타일**: 깊은 집중 선호, 타임블로킹 이미 사용 중
- **성격**: 생산성 도구 덕후, 데이터 기반 의사결정
- **스트레스 요인**: 산만한 환경, 비효율, 마감 압박

#### 고충점
1. 홈 오피스에서 집중 방해 요소 많음 (가족, 택배, 전화)
2. 프로젝트별 시간 추적이 번거로움 (청구서 작성용)
3. 카페에서 작업 시 타이머 소리 곤란
4. Toggl + Notion 따로 사용 중 → 통합 원함

#### 니즈
1. 조용한 타이머 (카페에서 사용)
2. 프로젝트별 시간 블록 관리
3. 주간/월간 생산성 리포트
4. 데스크톱 + 모바일 동기화

### 페르소나 3: 요리사 이민지 (Tertiary Persona)

#### 기본 정보
- **이름**: 이민지
- **나이**: 28세
- **직업**: 레스토랑 수셰프
- **근무지**: 강남 프렌치 레스토랑
- **근무 시간**: 오전 10시~오후 11시 (브레이크 타임 있음)

#### 고충점
1. 주방 소음이 커서 타이머 소리 안 들림
2. 손이 더러워서 스마트폰 터치 곤란
3. 여러 요리를 동시에 타이밍 맞춰야 함
4. 정확한 조리 시간 필수 (1분 차이로 맛 달라짐)

#### 니즈
1. **시각적 알림 강화**: 화면 플래시 + 진동
2. **음성 명령**: "타이머 5분 시작" (손 씻지 않고 조작)
3. **다중 타이머**: 동시에 3-4개 타이머 작동
4. **방수 기능**: 물기 있는 손으로도 사용 가능 (PWA)

#### 사용 시나리오
- 스테이크 굽기: 3분 → 뒤집기 → 2분
- 소스 조리: 15분 약불 → 화면 플래시로 알림
- 파스타 삶기: 7분 → 진동 알림

## 3. 가치제안 캔버스 (Value Proposition Canvas)

### 독서실 수험생 김지수를 위한 가치제안 캔버스

```
┌─────────────────────────────────────┐  ┌─────────────────────────────────────┐
│     고객 프로필 (Customer Profile)    │  │      가치 맵 (Value Map)              │
│                                     │  │                                     │
│  👤 고객 세그먼트: 독서실 수험생      │  │  📦 제품/서비스: TIMELOCK            │
└─────────────────────────────────────┘  └─────────────────────────────────────┘
```

#### 고객 프로필 (Customer Profile)

##### 📋 Jobs (고객이 해야 할 일)

**기능적 Jobs**
1. 하루 학습 시간을 30분~2시간 단위로 계획하기
2. 계획한 시간에 맞춰 공부 시작하고 종료하기
3. 순수 학습 시간과 휴식 시간 구분하기
4. 과목별 학습 시간 분배하기
5. 하루 학습 시간 목표 달성 여부 확인하기
6. 독서실에서 조용히 타이머 사용하기
7. 지난 주/월 학습 패턴 분석하기
8. 비효율적인 시간대 파악하고 개선하기

**사회적 Jobs**
1. 다른 수험생들에게 성실한 모습 보이기
2. 독서실 매너 지키기 (소음 없이 공부하기)
3. 스터디 그룹에서 학습량 공유하기

**감정적 Jobs**
1. 오늘 하루를 통제하고 있다는 느낌 받기
2. 계획을 지켰을 때 성취감 느끼기
3. 시간 낭비에 대한 죄책감 해소하기
4. 합격에 대한 자신감 얻기

**보조적 Jobs**
1. 학습 도구를 빠르게 세팅하기 (10초 이내)
2. 여러 앱을 오가지 않고 하나로 관리하기
3. 데이터를 안전하게 백업하기

##### 😰 Pains (고통/문제점)

**결과 기반 고통**
- 😤 계획은 10시간인데 실제로는 6시간만 공부함 → 목표 미달
- 😫 어제, 지난주와 비교해서 나아졌는지 알 수 없음 → 개선 불가
- 😓 어느 과목에 시간을 많이/적게 썼는지 모호함 → 시간 배분 실패
- 😱 시험 D-30인데 학습 진도가 느림 → 불안감 증가

**예상치 못한 고통**
- 😒 타이머 소리 나서 독서실에서 눈치 보임 → 사용 포기
- 😔 휴대폰 열었다가 SNS 보게 됨 → 시간 낭비
- 😩 여러 앱 사용하느라 데이터 정리 시간 소모 → 비효율
- 😵 타이머 울렸는데 못 들어서 시간 초과 → 계획 틀어짐

**장애물 및 위험**
- 🚫 Forest는 소리 알림만 있어서 독서실 부적합
- 🚫 종이 플래너는 실행 시간 자동 기록 안 됨
- 🚫 Notion은 타이머 기능 없음
- 🚫 Toggl은 계획 기능 없고 사후 기록만 가능
